92 ## About The Project
93 <div id="about-the-project"></div>
94 
95 This projects primary goal is to create a comfortable development environment for working with Markov Models,
96 as well as creating an end product which can be used for generating password wordlists using Markov Models.
97 
98 This project contains following sub-projects:
99 
100 - MarkovModel
101  - A versatile header-only template library for basic Markov Model structure.
102 - MarkovAPI
103  - A static/dynamic library built on MarkovModel, specialized to generate single-word lines.
104 - MarkovAPICLI
105  - A command line interface built on top of MarkovAPI
106 - Markopy
107  - A CPython extension wrapper for MarkovAPI, along with its own command line interface.
108 - MarkovPasswordsGUI
109  - A graphical user interface for MarkovAPI
110 - CudaMarkovAPI
111  - GPU-accelerated wrapper for MarkovAPI
112 - CudaMarkopy
113  - GPU-accelereted wrapper for CudaMarkovAPI
114 
115 
116 ### Possible Use Cases
117 <div id="possible-use-cases"></div>
118 
119 While main focus of the development has been towards random walk performance and password generation, underlying libraries could be used for other applications such as specific use cases of hidden markov models in bioinformatics and gene research.

122 ### Getting Started
123 <div id="getting-started"></div>
124 
125 If you'd just like to use the project without contributing, check out the releases page.
126 Latest minor release (0.8.x, 0.9.x) is even with main branch, and latest patch release (0.8.1, 0.8.2) is even with development branch.

128 
129 ### Releases
130 <div id="releases"></div>
131 
132 Releases are maintained automatically via github actions. Each push to the main branch will trigger a minor version release, while each accepted pull request into the development branch will trigger a patch version release.
133 
134 Pull requests to the development branch will also trigger a draft release only visible to the maintainers.
135 
136 Release files contain:
137 - libmarkov-{version}-{platform}.zip
138  - Depending on the platform, contains the libmarkov.so or markov.lib from that version.
139 - libcudamarkov-{version}-{platform}.zip
140  - Depending on the platform, contains the libcudamarkov.so or cudamarkov.lib from that version.
141 - markopy-{version}-{platform}-py{ver}.{extension}.zip
142  - Depending on the paltform, contains markopy.so or markopy.pyd. CPython extensions are compiled for specific versions.
143  If your python version is not supported by the releases, you can create an issue, or build it yourself using the python3.x-dev package.
144 - cudamarkopy-{version}-{platform}-py{ver}.{extension}.zip
145  - Depending on the paltform, contains cudamarkopy.so or cudamarkopy.pyd. CPython extensions are compiled for specific versions.
146  If your python version is not supported by the releases, you can create an issue, or build it yourself using the python3.x-dev package.
147 - models-{version}.zip
148  - Contains the latest models with the release version.
149  Contains base models (untrained), trained models, and language-specific models.

162 ### General prequisites
163 <div id="general-prequisites"></div>
164 
165 To build the simple core of this project, you'll need:
166 #### Linux
167 - CMake, preferably one of the latest versions.
168 - CXX compiler, preferably g++ or clang++ (LLVM 3.9+).
169 
170 #### Windows
171 - CMake, preferably one of the latest versions.
172 - CXX compiler, preferably msbuild(cl.exe) or clang++ (LLVM 3.9+).
173  Please note that mingw is not recommended as it is not officially supported by the nvcc.exe, and might not be linkable if you are building the CUDA components too.

215 ### CMake Configuration
216 <div id="cmake-configuration"></div>
217 
218 You can build this project with cmake.
219 
220 If you don't have prequisites for some of the projects set up, you can use the partial set up configuration to ignore those projects when setting the project up.
221 
222 If you do not meet the prequisites, you'll have to partially set up the CMake file (you cant use --target to build some of the targets, because configuration phase will fail too).
223 
224 Some examples for partially setting up and building the project are below.
225 
226 #### Build everything
227 <div id="build-all"></div>
228 
229 ```
230 $ cmake . -DPYTHON_VER=38 && cmake --build .
231 ```
232 
233 This will build all the libraries and executables. Requires python-dev, CUDA, QT5, QT5-Webview
234 
235 #### Build libraries only
236 <div id="build-libs"></div>
237 
238 ```
239 $ cmake . -DPARTIAL=1 -DB_LIBS=1 && cmake --build .
240 ```
241 
242 Only build basic libraries. Requires only CXX compiler.
243 
244 #### Build CUDA-accelerated libraries
245 <div id="build-cuda-libs"></div>
246 
247 ```
248 $ cmake . -DPARTIAL=1 -DB_CUDA=1 && cmake --build .
249 ```
250 
251 Build libraries along with cuda accelerated ones.
252 
253 #### Build python module & libraries
254 <div id="build-py-libs"></div>
255 
256 ```
257 $ cmake . -DPARTIAL=1 -DPYTHON_VER=39 && cmake --build .
258 ```
259 
260 Will build basic libraries and python modules.
261 
262 #### Build CUDA accelerated python module
263 <div id="build-cpy-libs"></div>
264 
265 ```
266 $ cmake . -DPARTIAL=1 -DPYTHON_VER=39 -DB_CUDA=1 && cmake --build .
267 ```
268 
269 Will build cudamarkopy.
270 
271 #### Build CUDA accelerated python module and the GUI
272 <div id="build-cpy-libs-and-gui"></div>
273 
274 ```
275 $ cmake . -DPARTIAL=1 -DPYTHON_VER=39 -DB_CUDA=1 -DB_GUI && cmake --build .
276 ```

283 ### Installing Dependencies
284 <div id="installing-deps"></div>
285 
286 ##### Windows
287 <div id="installing-deps-win"></div>
288 
289 - QT: Install [QT For Windows](https://doc.qt.io/qt-5/windows.html)
290 - Boost (program_options and python):
291  - Download Boost from [its website](https://www.boost.org/users/download/). Prefer one of the tested versions, 1.71.0 to 1.76.0
292  - Unzip the contents.
293  - Launch "Visual Studio Developer Command Prompt" (If you don't have this, properly set up the %PATH% variable for cl.exe)
294  - Move to the boost installation directory. Bootstrap libraries with your python version:
295  ```
296  .\bootstrap.bat --with-python=$(which python3.6) --with-python-version=3.6;
297  ```
298 
299  - Run `b2` to build the libraries.
300  ```
301  .\b2.exe --layout=system address-model=64 variant=release link=static runtime-link=shared threading=multi --with-program_options --with-python stage;
302  ```
303 
304 - Python: You can use the windows app store to download python runtime and libraries.
305 
306 
307 ##### Linux
308 <div id="installing-deps-lin"></div>
309 
310 - QT: Follow [this guide](https://wiki.qt.io/Install_Qt_5_on_Ubuntu) to install QT on Linux.
311  Alternatively, on ubuntu you can `sudo apt-get install qt5-qmake qtwebengine5-dev`
312 - Boost (program options and python):
313  - Download Boost from [its website](https://www.boost.org/users/download/). Prefer one of the tested versions, 1.71.0 to 1.76.0
314  - Unzip the contents.
315  - Move to the boost installation directory. Bootstrap libraries with your python version:
316  ```
317  ./bootstrap.sh --with-python=$(which python3.6) --with-python-version=3.6;
318  ```
319 
320  - Run `b2` to build the libraries.
321  ```
322  ./b2 variant=release link=static threading=multi --with-program_options install;
323  ./b2 --with-python --buildid=3.6 install;
324  ```
325 
326 - Boost (alternative)
327  - Use a package manager to install boost
328  ```
329  sudo apt-get install libboost-all-dev
330  ```
331 
332 - Python:
333  ```
334  sudo apt-get install python3-dev
335  ```

340 ## Known Common issues
341 <div id="known-issues"></div>
342 
343 ### Linux
344 #### Markopy - Python.h - Not found
345 Make sure you have the development version of python package, which includes the required header files.
346 Check if header files exist: `/usr/include/python*` or `locate Python.h`.
347 
348 If it doesn't, run `sudo apt-get install python3-dev`
349 
350 #### Markopy/MarkovAPI - *.so not found, or other library related issues when building
351 Run:
352 ```
353 ls /usr/lib/x86_64-linux-gnu/ | grep boost
354 ```
355 
356 and check the shared object filenames. A common issue is that lboost is required but filenames are formatted as libboost, or vice versa.
357 
358 Do the same for python related library issues, run:
359 ```
360 ls /usr/lib/x86_64-linux-gnu/ | grep python
361 ```
362 
363 to verify filename format is as required.
364 
365 If not, you can modify the makefile, or create symlinks such as:
366 ```
367 ln -s /usr/lib/x86_64-linux-gnu/libboost_python38.so /usr/lib/x86_64-linux-gnu/boost_python38.so
368 ```
369 
370 ### Windows
371 #### Boost - Bootstrap.bat "ctype.h" not found
372 - Make sure you are working in the "Visual Studio Developer Command Prompt" terminal.
373 - Make sure you have Windows 10 SDK installed.
374 - From VS developer terminal, run echo %INCLUDE%. If result does not have the windows sdk folders, run the following before running bootstrap (change your sdk version instead of 10.0.19041.0):
375 ```bat
376 set INCLUDE=%INCLUDE%;C:\Program Files (x86)\Windows Kits\NETFXSDK\4.8\include\um;C:\Program Files (x86)\Windows Kits\10\include\10.0.19041.0\ucrt;C:\Program Files (x86)\Windows Kits\10\include\10.0.19041.0\shared;C:\Program Files (x86)\Windows Kits\10\include\10.0.19041.0\um;C:\Program Files (x86)\Windows Kits\10\include\10.0.19041.0\winrt;C:\Program Files (x86)\Windows Kits\10\include\10.0.19041.0\cppwinrt
377 
378 set LIB=%LIB%;C:\Program Files (x86)\Windows Kits\10\lib\10.0.19041.0\ucrt\x64;C:\Program Files (x86)\Windows Kits\10\lib\10.0.19041.0\um\x64
379 ```
380 
381 #### Cannot open file "*.lib"
382 Make sure you have set the BOOST_ROOT environment variable correctly. Make sure you ran `b2` to build library files from boost sources.
383 
384 #### Python.h not found
385 Make sure you have python installed, and make sure you set PYTHON_PATH environment variable.

111 ## About The Project
112 <div id="about-the-project"></div>
113 
114 This projects primary goal is to create a comfortable development environment for working with Markov Models,
115 as well as creating an end product which can be used for generating password wordlists using Markov Models.
116 
117 This project contains following sub-projects:
118 
119 - MarkovModel
120  - A versatile header-only template library for basic Markov Model structure.
121 - MarkovAPI
122  - A static/dynamic library built on MarkovModel, specialized to generate single-word lines.
123 - MarkovAPICLI
124  - A command line interface built on top of MarkovAPI
125 - Markopy
126  - A CPython extension wrapper for MarkovAPI, along with its own command line interface.
127 - MarkovPasswordsGUI
128  - A graphical user interface for MarkovAPI
129 - CudaMarkovAPI
130  - GPU-accelerated wrapper for MarkovAPI
131 - CudaMarkopy
132  - GPU-accelereted wrapper for CudaMarkovAPI
133 
134 
135 ### Possible Use Cases
136 <div id="possible-use-cases"></div>
137 
138 While main focus of the development has been towards random walk performance and password generation, underlying libraries could be used for other applications such as specific use cases of hidden markov models in bioinformatics and gene research.

148 ### Releases
149 <div id="releases"></div>
150 
151 Releases are maintained automatically via github actions. Each push to the main branch will trigger a minor version release, while each accepted pull request into the development branch will trigger a patch version release.
152 
153 Pull requests to the development branch will also trigger a draft release only visible to the maintainers.
154 
155 Release files contain:
156 - libmarkov-{version}-{platform}.zip
157  - Depending on the platform, contains the libmarkov.so or markov.lib from that version.
158 - libcudamarkov-{version}-{platform}.zip
159  - Depending on the platform, contains the libcudamarkov.so or cudamarkov.lib from that version.
160 - markopy-{version}-{platform}-py{ver}.{extension}.zip
161  - Depending on the paltform, contains markopy.so or markopy.pyd. CPython extensions are compiled for specific versions.
162  If your python version is not supported by the releases, you can create an issue, or build it yourself using the python3.x-dev package.
163 - cudamarkopy-{version}-{platform}-py{ver}.{extension}.zip
164  - Depending on the paltform, contains cudamarkopy.so or cudamarkopy.pyd. CPython extensions are compiled for specific versions.
165  If your python version is not supported by the releases, you can create an issue, or build it yourself using the python3.x-dev package.
166 - models-{version}.zip
167  - Contains the latest models with the release version.
168  Contains base models (untrained), trained models, and language-specific models.

175

177

179 
180 ### Using Markopy/CudaMarkopy
181 <div id="using-markopy"></div>
182 
183 You can access basic operations from various model types using the python module, and if you are inexperienced with:
184 - Libmarkov and Libcudamarkov internals
185 - C++ code in general
186 - Importing and extending libraries in general
187 - Working with Python/C++ intermixed code
188 
189 We strongly recommend using the python module.
190 
191 While almost all of the python files provide their own entry point, you should use markopy.py or cudamarkopy.py depending on your preferences.
192 Please note that CUDA code will not run without an NVIDIA graphics card, and without CUDA runtime.
193 
194 markopy.py and cudamarkopy.py will let you select the model type you want to use with the -mt parameter. With each model, there are slightly different parameters available.
195 
196 For top level CLI selector (markopy.py and cudamarkopy.py)
197 ```
198 Model Mode selection choices:
199 usage: cudamarkopy.py [-mt MODEL_TYPE] [-h] [-ev EVALUATE] [-evt EVALUATE_TYPE]
200 
201 Python wrapper for MarkovPasswords.
202 
203 optional arguments:
204  -mt MODEL_TYPE, --model_type MODEL_TYPE
205  Model type to use. Accepted values: MP, MMX
206  -h, --help Model type to use. Accepted values: MP, MMX
207  -ev EVALUATE, --evaluate EVALUATE
208  Evaluate a models integrity
209  -evt EVALUATE_TYPE, --evaluate_type EVALUATE_TYPE
210  Evaluation type, model or corpus
211 
212  Sample runs:
213  markopy.py -mt MP generate trained.mdl -n 500 -w output.txt
214  Import trained.mdl, and generate 500 lines to output.txt
215 
216  markopy.py -mt MMX generate trained.mdl -n 500 -w output.txt
217  Import trained.mdl, and generate 500 lines to output.txt
218 
219  cudamarkopy.py -mt CUDA generate trained.mdl -n 500 -w output.txt
220  Import trained.mdl, and generate 500 lines to output.txt
221 
222 ```

347 #### Model selection
348 <div id="markopy-select"></div>
349 
350 You may use the -mt parameter from markopy.py or cudamarkopy.py to select a model type.
351 Allowed parameters are: MP, MMX, CUDA

354 
355 **Following are applicable for -mt MP mode:**
356 
357 ```
358 Python wrapper for MarkovPasswords.
359 
360 positional arguments:
361  mode Process mode. Either 'Train', 'Generate', or 'Combine'.
362  input Input model file. This model will be imported before starting operation.
363 
364 optional arguments:
365  -h, --help show this help message and exit
366  -o OUTPUT, --output OUTPUT
367  Output model file. This model will be exported when done. Will be ignored for generation mode.
368  -d DATASET, --dataset DATASET
369  Dataset file to read input from for training. Will be ignored for generation mode.
370  -s SEPERATOR, --seperator SEPERATOR
371  Seperator character to use with training data.(character between occurrence and value)
372  -t THREADS, --threads THREADS
373  Number of lines to generate. Ignored in training mode.
374  -v, --verbosity Output verbosity.
375  -b, --bulk Bulk generate or bulk train every corpus/model in the folder.
376  -w WORDLIST, --wordlist WORDLIST
377  Wordlist file path to export generation results to. Will be ignored for training mode

379  --max MAX Maximum length that is allowed during generation

484 #### Training
485 <div id="markopy-train"></div>
486 
487 If you do not have a custom corpus to train with, you may use one of the pre-trained models from the github releases.
488 
489 ```
490 python3 cudamarkopy.py train ../../../models/base-models/2gram.mdl -d ../../../datasets/graduation.corpus -s "\\t" -o test.mdl -vvvvvvvvv
491 ```
492 
493 #### Generation
494 <div id="markopy-generate"></div>
495 
496 If you have not trained a model, you may download on of the trained models (preferably models/trained/trained.mdl) from the releases and use it for generation.
497 
498 **Generating to a file**
499 ```
500 python3 cudamarkopy.py generate trained.mdl -mt MMX -n 5000 --min 6 --max 12 -w test.txt -vvvv
501 ```
502 
503 **Generating to stdout**
504 ```
505 python3 cudamarkopy.py generate trained.mdl -mt MMX -n 5000 --min 6 --max 12 --stdout
506 ```
507 
508 **Generating with CUDA model**
509 **WARNING** Do not use CUDA model unless you want to generate 500M+ lines. Prefer stdout mode with pipes instead of writing to disk whenever possible
510 
511 
512 **Using with Hashcat**
513 ```
514 python3 cudamarkopy.py generate trained.mdl -mt CUDA -n 5000 --min 6 --max 12 --stdout | hashcat -m 1400 hashes.txt -O
515 ```
516 
517 
518 **Use with hashcat, continue until terminated**
519 ```
520 python3 cudamarkopy.py generate trained.mdl -mt CUDA -n 5000 --min 6 --max 12 --stdout --infinite | hashcat -m 1400 hashes.txt -O
521 ```

659 ### Installing Dependencies

661 
662 ##### Windows
663 <div id="installing-deps-win"></div>
664 
665 - QT: Install [QT For Windows](https://doc.qt.io/qt-5/windows.html)
666 - Boost (program_options and python):
667  - Download Boost from [its website](https://www.boost.org/users/download/). Prefer one of the tested versions, 1.71.0 to 1.76.0
668  - Unzip the contents.
669  - Launch "Visual Studio Developer Command Prompt" (If you don't have this, properly set up the %PATH% variable for cl.exe)
670  - Move to the boost installation directory. Bootstrap libraries with your python version:
671  ```
672  .\bootstrap.bat --with-python=$(which python3.6) --with-python-version=3.6;
673  ```
674 
675  - Run `b2` to build the libraries.
676  ```
677  .\b2.exe --layout=system address-model=64 variant=release link=static runtime-link=shared threading=multi --with-program_options --with-python stage;
678  ```
679 
680 - Python: You can use the windows app store to download python runtime and libraries.

716 ## File Structure
717 <div id="file-structure"></div>
718 
719 You may chose to create your own model structures or corpus files. Following, is the reference for the current structure for them.
720 
721 ### Model
722 <div id="model-structure"></div>
723 
724 Model files are basically a list of edges in the model
725 
726 Format is `{left_node_content},weight,{right_node_content}\n`
727 
728 Example:
729 
730 ```
731 l,5,r
732 l,12,g
733 ...
734 ```
735 
736 There are additional requirements for a model for entry and termination nodes.
737 Entry nodes are represented with 0x00, and termination nodes are represented with 0xff
738 
739 It is expected (but not mandatory) to have an edge from starting node to all of the other nodes, and same applies for edges from each nodes to the termination nodes.
740 
741 If termination node has any edges that it is position on the left (meaning model file expectation is to traverse to another node after termination node) it will be loaded to the edges, but will be ignored during the random walk logic.
742 
743 ### Corpus
744 <div id="corpus-structure"></div>
745 
746 Corpus files are used to train models. You may chose to train your own model (preferably over an existing base model, like 2gram.mdl) to have your own generation style.
747 
748 Corpus file format is:
749 
750 `{occurrence}{seperator}{string}\n`
751 
752 - Occurrence is the weight associated with that string.
753 - Seperator is a single character seperator used to seperate occurrence with the string
754 - String is the password/string/sequence you want to add to the model.

758 ## Known Common issues
759 <div id="known-issues"></div>
760 
761 ### Linux
762 #### Markopy - Python.h - Not found
763 Make sure you have the development version of python package, which includes the required header files.
764 Check if header files exist: `/usr/include/python*` or `locate Python.h`.
765 
766 If it doesn't, run `sudo apt-get install python3-dev`
767 
768 #### Markopy/MarkovAPI - *.so not found, or other library related issues when building
769 Run:
770 ```
771 ls /usr/lib/x86_64-linux-gnu/ | grep boost
772 ```
773 
774 and check the shared object filenames. A common issue is that lboost is required but filenames are formatted as libboost, or vice versa.
775 
776 Do the same for python related library issues, run:
777 ```
778 ls /usr/lib/x86_64-linux-gnu/ | grep python
779 ```
780 
781 to verify filename format is as required.
782 
783 If not, you can modify the makefile, or create symlinks such as:
784 ```
785 ln -s /usr/lib/x86_64-linux-gnu/libboost_python38.so /usr/lib/x86_64-linux-gnu/boost_python38.so
786 ```
787 
788 ### Windows
789 #### Boost - Bootstrap.bat "ctype.h" not found
790 - Make sure you are working in the "Visual Studio Developer Command Prompt" terminal.
791 - Make sure you have Windows 10 SDK installed.
792 - From VS developer terminal, run echo %INCLUDE%. If result does not have the windows sdk folders, run the following before running bootstrap (change your sdk version instead of 10.0.19041.0):
793 ```bat
794 set INCLUDE=%INCLUDE%;C:\Program Files (x86)\Windows Kits\NETFXSDK\4.8\include\um;C:\Program Files (x86)\Windows Kits\10\include\10.0.19041.0\ucrt;C:\Program Files (x86)\Windows Kits\10\include\10.0.19041.0\shared;C:\Program Files (x86)\Windows Kits\10\include\10.0.19041.0\um;C:\Program Files (x86)\Windows Kits\10\include\10.0.19041.0\winrt;C:\Program Files (x86)\Windows Kits\10\include\10.0.19041.0\cppwinrt
795 
796 set LIB=%LIB%;C:\Program Files (x86)\Windows Kits\10\lib\10.0.19041.0\ucrt\x64;C:\Program Files (x86)\Windows Kits\10\lib\10.0.19041.0\um\x64
797 ```
798 
799 #### Cannot open file "*.lib"
800 Make sure you have set the BOOST_ROOT environment variable correctly. Make sure you ran `b2` to build library files from boost sources.
801 
802 #### Python.h not found
803 Make sure you have python installed, and make sure you set PYTHON_PATH environment variable.
